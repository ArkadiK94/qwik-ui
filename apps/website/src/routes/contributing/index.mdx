# Contributing to the Qwik Framework Component Library

Thank you for considering contributing to the Qwik Framework Component Library! This document outlines some guidelines for contributing to the project.

## Getting Started

Before contributing, make sure you have the following installed:

- Node.js (version 14 or later)
- Yarn

To get started, clone the repository:

```
git clone https://github.com/qwik-framework/component-library.git
```

Then, install the dependencies:

```
cd component-library
yarn install
```

You can then start the development server:

```
yarn start
```

This will start a development server at `http://localhost:3000` where you can view the components in action.

## Creating a Component

To create a new component, you can use the `yarn new` command:

```
yarn new Button
```

This will create a new component named `Button` in the `src/components` directory. You can then edit the component code in the newly created `Button.tsx` file.

Make sure to follow the Qwik Framework Component Library Design System guidelines when creating a new component.

## Testing

Make sure to test your changes before submitting a pull request. You can run the tests using:

```
yarn test
```

This will run the Jest test suite.

## Submitting a Pull Request

When submitting a pull request, make sure to follow these guidelines:

- Create a descriptive title that summarizes your changes.
- Include a brief description of your changes in the pull request description.
- Make sure all tests pass.
- Make sure your code follows the Qwik Framework Component Library Code Style guidelines.
- Make sure to update the documentation if necessary.

## Conclusion

Thank you for contributing to the Qwik Framework Component Library! If you have any questions or concerns, please don't hesitate to reach out to the maintainers.
